# LoveKey Build 57 實機與付款驗收清單

## 指定版本

- App：LoveKey 1.0.4
- TestFlight Build：57
- 來源 commit：`66bd55feffe83231e7f5f7bc413db2cb0a96db29`
- 狀態：`VALID`，已加入 `Internal Testing`

不要使用 Build 55 或 Build 56 做最終驗收。

## 測試前準備

1. 在 iPhone 刪除舊版 LoveKey。
2. 從 TestFlight 全新安裝 Build 57。
3. 使用專用測試帳號及測試對話，不要使用正式客戶資料。
4. 付款只使用 Apple Sandbox／TestFlight 環境，不得真實扣款。
5. 每個失敗畫面都截圖，並記錄裝置、iOS 版本、時間與操作步驟。

## A. App 與登入

- [ ] 冷啟動可進入 App，不閃退、不白屏。
- [ ] 完成 onboarding；關閉重開後不會無故重跑。
- [ ] Email 註冊、登入、登出、重新登入正常。
- [ ] Google 登入成功與取消流程正常。
- [ ] Apple 登入成功與取消流程正常。
- [ ] 關閉重開後登入狀態符合預期。

## B. 第三方鍵盤核心流程

- [ ] 依教學加入 LoveKey 鍵盤並開啟「允許完整取用」。
- [ ] 在 LINE 複製一則測試訊息，切到 LoveKey，選模式與語氣，生成一則回覆並填入。
- [ ] 在 Instagram 重複同一流程。
- [ ] 在 iMessage 重複同一流程。
- [ ] 空白、短句、長句、多行、emoji、網址與特殊符號不會造成當機。
- [ ] 快速連點生成／重試不會重複送出請求，也不會讓整個鍵盤閃爍。
- [ ] 切換其他鍵盤、返回 LoveKey、切換前景／背景後仍可使用。
- [ ] 未開完整取用時顯示清楚提示，不會假裝生成成功。

## C. AI 回覆品質與錯誤處理

- [ ] 「哈哈」能自然接回話題，不尷尬、不過度曖昧。
- [ ] 「你平常都在幹嘛？」能帶出可延伸話題。
- [ ] 「好喔」改寫後自然、有溫度、不油膩。
- [ ] 咖啡／散步／貓的破冰回覆自然。
- [ ] 週末咖啡邀請舒服、不施壓。
- [ ] 「最近有點忙」能尊重對方並自然延伸。
- [ ] 對方表示不舒服時先道歉降溫，再澄清，不辯解或操控。
- [ ] 斷網、慢網、逾時、服務拒答時顯示可理解的繁中錯誤，恢復網路後可重試。

## D. 週／年／永久商品與權益

- [ ] Paywall 顯示週、年、永久三項商品。
- [ ] 價格使用 iPhone App Store 帳號所在地的當地貨幣，不是寫死美元。
- [ ] Apple Sandbox 週訂閱成功後，App 與鍵盤立即同步 Pro。
- [ ] Apple Sandbox 年訂閱成功後，App 與鍵盤立即同步 Pro。
- [ ] Apple Sandbox 永久會員成功後，App 與鍵盤立即同步 Pro。
- [ ] 取消付款、付款失敗與連續點擊不會誤解鎖。
- [ ] 關閉重開、登出登入後權益正確。
- [ ] 刪除重裝後「恢復購買」可恢復既有權益。
- [ ] 沒有購買紀錄時，「恢復購買」顯示清楚提示。

## E. 顯示與易用性

- [ ] 一般字體與大字體不截字、不重疊。
- [ ] 小尺寸 iPhone、一般尺寸與 Pro Max 至少各驗證一台或一種等效尺寸。
- [ ] Safe Area、Dynamic Island 與鍵盤高度不遮擋主要按鈕。
- [ ] VoiceOver 可讀出主要控制項名稱。
- [ ] 目前產品固定淺色時，深色系統設定下仍須清楚可讀；若要真正深色模式，另開 UI 任務，不在本次自動修改。
